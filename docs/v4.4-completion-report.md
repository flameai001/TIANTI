# TIANTI V4.4 完成报告

- 项目名称：`TIANTI Web`
- 报告日期：`2026-04-29`
- 版本标识：`v4.4`

## 1. 结论

TIANTI `v4.4` 已完成本轮需求实现与验证。

本次版本围绕四条主线交付：

- 公开页活动阵容展示规则调整：已确认达人显示备注，待确认达人显示黄色 `UNCONFIRMED`。
- 达人详情历史出现优化：没有现场档案角色信息时不再显示占位文案。
- 后台活动录入流程拆分：添加达人、添加现场档案均改为独立弹窗，并补齐多日活动录入便利性与校验。
- 后台天梯展示调整：梯度改为横向单行展示，达人卡片压缩为自适应昵称 chip。

本轮未引入数据库结构变更，不需要迁移。

## 2. 关键交付

### 2.1 公共阵容展示与达人历史

- 首页活动卡、活动列表卡、活动详情阵容已统一展示规则。
- `confirmed` 状态达人：
  - 显示阵容备注 `lineup.note`。
  - 备注为空时不渲染说明行。
- `pending` 状态达人：
  - 显示黄色 `UNCONFIRMED`。
  - 不展示信息来源或备注。
- 达人详情页“历史出现”已取消 `暂未录入角色 / 作品 / 游戏` 占位文案。
- 当现场档案缺少角色、作品、游戏等详情时，历史出现条目保留活动信息，详情行留空。

涉及文件：

- `src/components/site/event-card.tsx`
- `src/app/(public)/events/[slug]/page.tsx`
- `src/app/(public)/talents/[slug]/page.tsx`

### 2.2 后台活动阵容弹窗

- 后台活动页新增可复用弹窗壳，用于承载独立录入流程。
- “添加达人”已改为独立弹窗。
- 支持选择达人、状态、日期与备注。
- 多日活动下支持一次为同一达人勾选多个日期，并为每个日期单独填写备注。
- 同一达人同一天已存在阵容时会被禁用，避免重复添加。
- 状态为“已确认”时隐藏并清空“信息来源”。
- 状态为“待确认”时显示并保存“信息来源”。
- 既有阵容行仍保留内联编辑能力。

涉及文件：

- `src/components/admin/archive-manager-v3.tsx`
- `src/components/admin/archive-manager-utils.ts`

### 2.3 后台现场档案弹窗与校验

- “添加现场档案”已改为独立弹窗。
- 支持一次新增多条现场记录，每条记录可独立添加、删除、填写。
- 现场档案达人下拉只包含当前活动已保存阵容里的达人。
- 多日活动下，档案日期只允许选择该达人已参与阵容的日期。
- 若活动信息或阵容存在未保存改动，添加现场档案前会提示先保存活动信息。
- 批量导入现场档案同样受未保存改动提示约束。

涉及文件：

- `src/components/admin/archive-manager-v3.tsx`
- `src/components/admin/archive-manager-utils.ts`
- `src/modules/admin/mutations.ts`

### 2.4 后台天梯横向梯度

- 后台天梯页从三列卡片网格调整为纵向梯度列表。
- 每个梯度只占一整行。
- 每行左侧保留梯度名称输入与操作按钮。
- 每行右侧为横向排列的达人 chip 列表。
- 达人 chip 只显示昵称。
- chip 使用自适应宽度，尽量缩小卡片尺寸。
- 当一行显示不下时使用浏览器原生横向滚动条和触控横滑。
- 空梯度保留行内虚线 drop zone。
- 保留原有拖拽排序、未入榜池、快速加入、保存逻辑和公开天梯同步能力。

涉及文件：

- `src/components/admin/ladder-manager.tsx`

### 2.5 服务端规则与数据规范化

- `saveEvent` 保存活动阵容时：
  - 已确认达人会将 `source` 规范化为空。
  - 待确认达人保留 `source`。
- `saveArchive` 保存现场档案时：
  - 校验档案达人必须存在于当前活动阵容。
  - 多日活动下校验档案日期必须匹配该达人在阵容中的参与日期。
- 后台编辑草稿归一化逻辑同步处理确认状态下的信息来源清空。

涉及文件：

- `src/modules/admin/mutations.ts`
- `src/components/admin/archive-manager-utils.ts`
