# TIANTI v4 全站体验重构设计文档

- 日期：`2026-04-16`
- 状态：`设计稿待审`
- 分支：`codex/tianti-v4`

## 1. 摘要

`v4` 是 TIANTI 的一次全站体验重构。

这次不是轻量视觉翻新，也不是业务逻辑重写。核心目标是在一套共享的亮色设计系统之上，重建公开站与后台 UI，让产品具备更强的层级、更清晰的信息架构、更成熟的交互完成度，以及真正面向用户的专业文案表达，而不是带有实现过程痕迹的说明式语言。

本次选定方向如下：

- 整体气质：`Premium Product`（高端产品感）
- 次级风格影响：`Gallery Editorial`（策展展示感）
- 视觉基调：亮色、现代、高级、克制
- 文案语气：公开站更偏策展与展示，后台更偏直接和产品化

## 2. 设计定义

`v4` 被定义为一次同时覆盖公开站和后台工作台的全体验重构。

包含内容：

- 重建视觉语言
- 重组页面结构与信息架构
- 重写用户可见文案
- 提升动效与交互完成度
- 将公开站与后台统一到一套一致的产品语言中

不包含内容：

- 重构数据模型
- 修改权限模型
- 重写 API 形状
- 新增业务内容域
- 超出布局与体验重组范围的大规模流程改写

## 3. 目标

### 3.1 核心目标

- 将公开站重构为一个亮色、现代、高级、适合展示与浏览的内容站。
- 将后台重构为一个更成熟的工作台，具备更清晰的导航、更稳定的布局秩序和更专业的反馈状态。
- 用更轻、更干净、更有设计感的整体语言替换当前偏深色、偏说明型的呈现方式。
- 删除带有版本过程、开发过程或 vibe-coding 痕迹的界面文案。
- 让 TIANTI 看起来像一个完整成熟的产品，而不是若干共享数据的独立模块拼接体。

### 3.2 体验目标

- 第一眼就能传达出品质感和明确意图。
- 公开站在更好看的同时，也变得更适合浏览和跳转。
- 后台在更统一的同时，也变得更稳、更清楚、更可信赖。
- 动效必须有存在感，但始终让位于可读性和可用性。

## 4. 非目标

除非后续实现中出现不可避免的小依赖，否则以下内容明确不属于 `v4` 范围：

- 数据库结构重构
- 认证与会话体系重构
- 新内容类型
- 新权限角色
- 新的平台绑定
- 改变核心业务模型的大规模流程重写

`v4` 是一次体验重构，而不是平台重写。

## 5. 产品范围

`v4` 同时覆盖公开站和后台。

### 5.1 公开站

- `/`
- `/talents`
- `/talents/[slug]`
- `/events`
- `/events/[slug]`
- `/ladder`
- `/search`
- 共享头部、页脚、导航、页面壳层、空状态与通用分区模式

### 5.2 后台

- `/admin/login`
- `/admin`
- `/admin/talents`
- `/admin/archives`
- `/admin/ladder`
- 共享后台导航、总览布局、列表区域、编辑区域、保存反馈、上传区域、空状态与状态提示

## 6. 信息架构

### 6.1 公开站导航

公开站一级导航应保持收敛：

- 首页
- 达人
- 活动
- 天梯
- 搜索

公开站不应继续用长段说明型文案解释产品结构，而应像一个清晰、明确的品牌入口，把用户引导到已有内容域中。

### 6.2 公开站页面职责

#### 首页

首页应从“功能解释页”转为“品牌入口 + 内容入口”页面。

建议结构为：

1. 强品牌 Hero
2. 精选入口
3. 最新或重点内容
4. 搜索 / 浏览深化入口

Hero 的职责是传达品牌以及 TIANTI 所承载的内容类型，不应再出现实现说明、版本叙事或内部产品解释。

#### 达人列表页

达人列表页应成为一个以展示为优先的浏览页面：

- 更强的封面表现
- 更清楚的筛选与排序呈现
- 更好的卡片节奏与视觉呼吸感
- 更高级的浏览与扫描体验

#### 活动列表页

活动列表页应成为一个以内容为导向的浏览页面：

- 突出时间、城市、场馆、阵容与档案上下文
- 让筛选区更像成熟产品，而不是一组表单
- 在保持发现价值的同时提高清晰度

#### 达人详情页

达人详情页应成为结构化的展示型人物页：

- Hero 区
- 身份与公开资料摘要
- 以图像为主导的内容区
- 相关内容区
- 更清晰的主次信息层级

#### 活动详情页

活动详情页应成为结构化的专题型内容页：

- 更强的活动摘要区
- 更统一的阵容与档案区
- 更合理的图像层级
- 更明确的阅读路径，从概览进入档案，再进入相关内容

#### 天梯页

天梯页应更像一个 curated ranking 页面，而不只是功能性列表展示页。

#### 搜索页

搜索页应继续保留统一检索入口，但结果呈现应更产品化、更易扫描。

### 6.3 后台导航

后台应从“多个页面集合”转为统一工作台。

后台主导航建议收敛为：

- 工作总览（Overview）
- Talents
- Events & Archives
- Ladder

`Events` 与 `Archives` 在内部仍可保留现有业务边界，但 UI 上应被呈现为一个统一工作区。

### 6.4 后台页面职责

#### 工作总览（Overview）

后台首页应成为真正的工作台总览页，包含：

- 最近动态
- 重要快捷入口
- 有用的状态摘要
- 当前工作上下文

#### Talents

达人管理页应更像现代内容管理工作区，列表、编辑、图片与状态反馈使用统一语言。

#### Events & Archives

活动档案工作区仍然是事件与档案录入的操作中心，但布局应更清晰：

- 左侧承担上下文与导航
- 右侧承担主编辑区
- 保存反馈与区块边界更明确

#### Ladder

天梯管理页应继承后台整体设计系统与布局纪律。

## 7. 视觉系统

### 7.1 视觉主张

TIANTI `v4` 应呈现为一个亮色、高级、带有编辑感克制的成熟产品。

它不应看起来像：

- 深色电影感站点
- 过度装饰的视觉实验
- 强模板感作品
- 泛 SaaS 卡片拼盘
- 为了时尚感而牺牲可用性的杂志化页面

### 7.2 色彩方向

设计系统应切换到亮色基底：

- 背景：干净的白色到浅冷灰
- 核心表面：白色与近白色
- 主强调色：一条蓝系品牌线，用于操作、导航、激活状态与重点提示
- 次级暖色：只在公开站局部氛围中少量使用

暖色展示层可以存在于公开站，但不应扩展成后台里的第二条完整系统色。

### 7.3 字体方向

字体需要建立比当前系统更清晰的层级。

规则：

- 一套高质量 Sans 字体，用于正文、界面与后台
- 一套克制的 Display 字体，用于公开站 Hero 与部分标题
- 公开站标题允许更有品牌感和展示感
- 后台标题应更偏产品化和工具化
- 正文始终以可读性优先，而不是风格表达优先

当前字体组合不默认保留到 `v4`。

### 7.4 表面与容器

卡片仍可使用，但不应继续主导整体语言。

系统更应依赖：

- 分区
- 留白
- 编组
- 分隔
- 只有在容器本身有意义时才使用承载式表面

设计规则：

- 更充足的间距
- 更精致的圆角
- 更轻的阴影
- 更干净的边框
- 更克制但更成熟的层叠关系

全站应避免退化成一面风格完全相同的浮层卡片墙。

### 7.5 图标

图标应保持干净、简洁，并与整体产品气质一致：

- 线性简洁图标
- 不做装饰性图标效果
- 不使用过度插画化或玩具感图标体系

## 8. 交互与动效

动效属于这次重构的一部分，需要能被感知，但绝不能喧宾夺主。

### 8.1 公开站动效

公开站建议包含：

- Hero 入场过渡
- 内容分区滚动露出
- 卡片与链接的精致 hover / focus 反馈
- 少量图像或表面的动态细节，用来增强完成度

### 8.2 后台动效

后台动效应更克制、更偏工具性质：

- 保存成功反馈
- 局部面板切换
- hover / focus 状态
- 状态变化反馈
- loading 与刷新过渡

### 8.3 动效规则

- 动效必须强化层级或反馈
- 动效必须在移动端依然流畅
- 动效不得拖慢后台核心操作
- 动效必须可降级，在 reduced motion 等场景下依然成立

## 9. 文案策略

### 9.1 公开站文案

公开站文案应更像策展式展示语言，短、准、克制、可公开呈现。

规则：

- 句子短而明确
- 面向访客与品牌表达
- 不做版本历史叙述
- 不做实现过程说明
- 除非有助于访客理解内容，否则不写内部产品解释

需要删除的文案类型包括：

- 版本过程型描述
- 面向开发者的说明
- 类似“v3 把 X 改成 Y”这样的文本
- 任何带有 vibe-coding 过程痕迹的界面叙述

### 9.2 后台文案

后台文案应更直接、更实用、更产品化。

规则：

- 先说明位置和上下文
- 先说明动作和结果
- 标签清晰
- 状态反馈清晰
- 不写气氛化空话

公开站与后台应共享同一个产品身份，但不共享同一种句式风格。

## 10. 页面级方向

### 10.1 首页

首页改造方向：

- 替换当前偏说明型的 Hero
- 建立更强的品牌 poster-like Hero
- 用精选入口替代系统解释型文字
- 用更有节奏的方式展示重点或最新内容
- 用清晰的搜索 / 浏览深化入口收尾

### 10.2 达人列表页

达人列表页改造方向：

- 强化封面驱动的浏览体验
- 现代化筛选区的位置与样式
- 优化结果区节奏和层级
- 让卡片更高级、更少模板感

### 10.3 活动列表页

活动列表页改造方向：

- 优化筛选区与结果区的关系
- 降低表单感的视觉重量
- 提高活动元数据的扫描效率
- 强化阵容与档案的入口提示

### 10.4 达人详情页

达人详情页改造方向：

- 建立更强的 Hero 区
- 重组摘要、公开资料、图像内容与相关内容
- 提高阅读节奏与视觉层级

### 10.5 活动详情页

活动详情页改造方向：

- 建立更强的摘要与 Hero 区
- 更好地组织阵容、日期分组、档案内容与相关内容
- 形成更明确的阅读顺序

### 10.6 搜索页

搜索页改造方向：

- 更清楚的结果分组
- 更产品化的筛选与结果交互
- 更好的视觉扫描路径

### 10.7 天梯页

天梯页改造方向：

- 更强的排名呈现感
- 更清晰的梯度与条目层级
- 更完整的公开页质感

### 10.8 后台工作总览

后台总览页改造方向：

- 从静态摘要卡片改为真正的 workspace 总览页
- 提高最近动态与快捷入口的层级
- 降低“盒子拼盘 dashboard”的感觉

### 10.9 后台 Talents

后台达人管理页改造方向：

- 统一列表与编辑语言
- 改进图片区与状态反馈
- 让间距、结构与整体工作台语言一致

### 10.10 后台 Events & Archives

后台活动档案工作区改造方向：

- 保留现有核心流程逻辑
- 重做布局，让工作区关系更清楚
- 提升区块层级
- 提升保存状态与上传状态的可读性
- 让左侧导航区与右侧编辑区更有设计完成度

### 10.11 后台 Ladder

后台天梯管理页改造方向：

- 与后台其它页面统一控件、间距、层级与保存反馈

## 11. 实施策略

`v4` 建议按四个阶段推进。

### 11.1 第一阶段：Foundation

先建立共享设计基础层：

- design tokens（设计令牌）
- 色彩
- 字体
- 间距
- 圆角
- 阴影
- 边框
- 页面壳层
- 按钮
- 输入框
- 筛选器
- 标签
- 空状态
- 状态提示
- 共享动效原语

这一阶段还需要清理当前对深色全局样式的深度依赖。

### 11.2 第二阶段：Public

将公开站作为一个整体完成重构：

- 首页
- 达人列表与详情
- 活动列表与详情
- 天梯
- 搜索
- 共享公开站导航与页脚

### 11.3 第三阶段：Admin

将后台工作台作为一个整体完成重构：

- 登录页
- 总览页
- 达人管理
- 活动与档案工作区
- 天梯管理
- 共享后台导航与反馈组件

### 11.4 第四阶段：Polish

最后进行全站收口：

- 文案清理
- 动效补齐
- 响应式细修
- hover / focus / loading / error / empty 状态检查
- 一致性巡检
- 视觉 QA
- 回归验证

## 12. 风险与护栏

### 12.1 主要风险

- 重构最终只变成视觉换皮
- 公开站和后台看起来像两个产品
- 动效沦为装饰噪音
- 公开站更好看了，但阅读效率下降
- 后台更好看了，但操作效率下降
- 文案清理只完成一部分，导致风格不统一

### 12.2 控制原则

- 先做共享设计系统，再做逐页重构
- 不允许无控制地扩展到 schema 或 API 重写
- 公开站优先展示与浏览
- 后台优先效率与清晰度
- 所有重写文案都必须真正面向用户

## 13. 验证与验收

### 13.1 验证方式

本次重构应通过以下方式验证：

- 桌面、平板、手机的视觉回归
- hover、focus、筛选、跳转、保存、上传、空状态、错误状态的交互回归
- 图片与动效带来的性能回归
- 后台核心操作链路的可用性回归
- 基于现有 lint、test、build、smoke、E2E 的自动化回归

### 13.2 验收标准

当以下条件成立时，可认为 `v4` 成功：

- 全站看起来明确像一次完整升级，而不是小修小补
- 公开站具备现代、亮色、高级、适合展示的气质
- 后台具备统一、成熟、专业的工作台体验
- 公开站与后台清楚地属于同一个产品家族
- 所有带有实现历史或 vibe-coding 痕迹的界面文案都被移除
- 动效可被感知，但始终克制
- 没有引入明显的浏览或后台核心操作回归

## 14. 最终决策记录

以下决策是在 brainstorming 过程中明确确认过的，并构成当前批准范围：

- `v2` 和 `v3` 视为已经闭环，即使没有最终总结报告
- `v4` 聚焦全 UI 与全体验重构
- 公开站与后台同等重要
- 允许调整页面结构与信息架构
- 文案策略为：公开站偏策展，后台偏专业产品
- 选定的执行深度是 `Full Experience Redesign`（全体验重构）
- 选定的视觉方向是 `Premium Product`，吸收 `Gallery Editorial` 的展示感
