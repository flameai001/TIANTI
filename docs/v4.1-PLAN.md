# TIANTI v4.1 Plan

- 版本目标：统一活动时间语义，移除手动未来/已结束维护，把首页、达人、活动、搜索、天梯和后台工作区收口到同一套读模型。
- 状态规则：活动状态按 `Asia/Shanghai` 的当日日期自动推导为 `future / past / undated`；同日仍算未来，无码日期不进入依赖时间状态的板块和筛选。
- 公开站点：
  - 首页删除搜索板块和编辑视角数量统计卡。
  - 未来活动按阵容规模优先显示，最多 2 个，无内容时隐藏。
  - 达人列表新增“有行程”筛选。
  - 达人详情移除相关活动/相关达人，新增“即将参与 / 历史出现”分页卡片。
  - 活动列表新增编辑者筛选。
  - 活动详情移除底部相关区块，档案卡改为“已认出 / 已集邮”。
  - 搜索页删除 scope 下拉，只保留搜索框下方切换按钮。
  - 天梯页删除梯度数量卡，改为单卡展示各梯度人数。
- 后台：
  - 总览删除常用入口卡片，昵称卡片移到底部。
  - 导航“活动与档案”改成“活动”。
  - 活动工作区删除单条状态选择与批量状态修改。
  - 档案录入允许现场图为空，“是否有合照”改为“是否有集邮”。
  - 裁剪弹层和残留暗色样式调整为与 v4 亮色后台一致。
- 数据结构：
  - `TalentRepresentation.assetId` 可空。
  - `ArchiveEntry.sceneAssetId` 可空。
  - 保留数据库 `events.status` 字段做兼容，但前台查询与后台展示不再信任手动值。
- 验证要求：
  - `npm run lint`
  - `npm test`
  - `npm run build`
  - `npm run test:e2e`
  - `npm run test:e2e:smoke`
