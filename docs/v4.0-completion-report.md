## 3. 关键实现说明

本轮保持了 `v4.0-PLAN` 的核心约束：

- 没有重构数据库结构
- 没有修改权限模型
- 没有重写现有公开站 query 参数契约
- 没有改动后台现有 talent/event/archive/ladder 的核心 payload 形状

补充说明：

- “编辑者昵称修改”使用的是现有 `editors.name` 字段，没有新增数据库字段或迁移
- 本轮新增的唯一后台接口是 `/api/admin/editor`，用于保存当前编辑者显示昵称
