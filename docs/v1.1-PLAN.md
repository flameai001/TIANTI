# TIANTI V1.1 活动档案收口规划

## 摘要
- 前台把“未来行程”和“活动档案”收口为一个一级栏目“活动档案”，以 `/events` 作为唯一主入口。
- 后台把“活动”和“活动档案”收口为一个一级栏目“活动档案”，以 `/admin/archives` 作为唯一主入口。
- 底层数据模型保持不变，继续保留共享活动数据与编辑人私有档案数据的分层；V1.1 只做信息架构、路由、筛选、页面编排和文案收口，不做数据库迁移。

## 关键改动
### 前台
- 站点导航、页脚、首页入口统一移除“未来行程”一级导航，只保留“活动档案”。
- `/events` 吸收当前 `/schedule` 的能力，成为统一活动列表页，支持：
  - 关键词
  - 城市
  - 达人
  - 时间范围
  - 活动状态
  - 参与状态
- 首页保留“近期未来活动”内容块，但所有入口改为跳到 `/events?eventStatus=future`，不再跳 `/schedule`。
- `/events/[slug]` 继续作为活动详情页，统一承接未来活动和已结束活动；当未来活动还没有任何档案记录时，必须显示明确空状态，不允许档案区块直接留空。
- `/schedule` 改为兼容跳转页，跳到 `/events` 并自动带上未来活动筛选。

### 后台
- `/admin/archives` 改为统一活动工作台：
  - 左侧为活动列表与“新建活动”
  - 右侧上半区为共享活动信息与达人阵容编辑
  - 右侧下半区为“我的档案”编辑与素材上传
- `/admin/events` 改为兼容跳转到 `/admin/archives`。
- 选中“未保存的新活动”时，只允许先保存活动基础信息；保存成功并拿到 `eventId` 后，再开启“我的档案”编辑区。
- 同一路由内保留两套独立保存动作：
  - `保存活动信息`
  - `保存我的档案`
- 删除活动仍沿用现有保护规则：只要任一编辑人的档案已关联该活动，就禁止删除。

### 接口与类型
- 前台活动列表的规范化查询参数改为：
  - `eventStatus`
  - `participationStatus`
  - `q`
  - `city`
  - `talent`
  - `from`
  - `to`
- `EventFilters` 内部字段从 `status` 明确改名为 `eventStatus`，避免与参与状态混淆。
- 后台统一页面用 `event=<eventId>` 保存当前选中的活动，保证刷新、保存、重定向后仍停留在同一活动。
- 继续保留现有写接口与数据表：
  - `/api/admin/events`
  - `/api/admin/events/[id]`
  - `/api/admin/archives`
  - `events / event_lineup / editor_archives / archive_entries`
- 兼容规则：
  - 旧 `/events?status=future|past` 继续可读，内部映射到 `eventStatus`
  - 旧 `/schedule?status=confirmed|pending` 在跳转时映射到 `participationStatus`

## 实施顺序
1. 先收口前台路由与筛选参数，完成 `/events` 合并与 `/schedule` 兼容跳转。
2. 再合并后台活动编辑台，完成 `/admin/archives` 单页工作台与 `/admin/events` 兼容跳转。
3. 最后统一导航文案、首页入口、空状态、测试与文档说明。

## 测试方案
- 前台：
  - `/events` 默认可浏览全部活动，未来/已结束状态可见。
  - `/events?eventStatus=future&participationStatus=confirmed` 能覆盖当前“未来行程”核心场景。
  - `/schedule` 跳转正确，并保留 `q/city/talent/from/to`，同时把旧 `status` 正确映射为 `participationStatus`。
  - 无档案的未来活动详情页显示明确空状态；有档案的活动详情页继续正常展示现场记录。
- 后台：
  - 在 `/admin/archives` 内可新建活动，首次保存后可继续在同页录入我的档案。
  - 在同一活动页内分别保存活动信息和我的档案时，不丢失当前选中活动。
  - `/admin/events` 会跳到 `/admin/archives`。
  - 已有关联档案的活动删除仍被拦截。
- 回归：
  - 更新 E2E 用例中所有 `/schedule`、`/admin/events`、旧页面标题与按钮文案。
  - 继续跑通 `lint`、单测、`build`、E2E 和移动端浏览冒烟。

## 假设与默认
- V1.1 不合并数据库表，不改变“共享活动 + 私有档案”的权限边界。
- `docs/v1-completion-report.md` 作为 V1 历史存档不重写；V1.1 说明应新增独立文档或补充 README/设计说明。
- 前台统一对外名称使用“活动档案”；“未来活动/已结束活动”只作为筛选或视图状态存在，不再作为一级栏目。
